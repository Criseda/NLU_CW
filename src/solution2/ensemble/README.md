# Transformer Ensemble Pipeline (Solution 2)

This directory contains the pipeline for fusing multiple transformer models using a late-fusion (soft-voting) strategy. 

## Architectural Overview
Instead of combining embeddings or training a complex meta-classifier, this ensemble takes the raw classification probabilities generated by each individual fine-tuned model and merges them via a weighted average.

The ensemble is heavily optimized around our best performing cross-encoder (checkpoint `8180`).

### Models & Optimal Weights
The fusion logic (found in `predict_ensemble.py`) assigns the following weights to maximize the Macro F1 score:

1. **DeBERTa Model 8180:** 50% voting power (Strongest individual model)
2. **DeBERTa Model 803:** 40% voting power (Runner-up)
3. **MiniLM-L6 (Small Model):** 10% voting power (Added for representation diversity)
4. *DeBERTa Model 7931:* 0% (Silenced to prevent dragging the score down)
5. *Original DeBERTa Big:* 0% (Silenced to prevent dragging the score down)

## Pipeline Execution
You do not need to run the python scripts individually. A shell script handles the end-to-end integration:

```bash
bash run_pipeline.sh
```

**Step-by-step Execution:**
1. Triggers the inference script (`src.solution2.big_model.predict`) on the `dev` dataset using the DeBERTa `803` checkpoint.
2. Triggers the inference script again for the `8180` checkpoint. 
3. Combines their output CSV probabilities with the existing MiniLM probabilities via the `predict_ensemble.py` script.
4. Outputs the final prediction CSV (`ENSEMBLE_AV_dev.csv`).
5. Evaluates the predictions using the official local scorer.
